O trabalho foi implementado com sucesso utilizando Docker Compose, contendo cinco contêineres: um contêiner com Nginx, três contêineres com WordPress e um contêiner com MySQL. O Nginx foi configurado como balanceador de carga, distribuindo as requisições entre as três instâncias do WordPress. As instâncias do WordPress compartilham o mesmo banco de dados MySQL, permitindo acesso ao mesmo conteúdo. Também foi configurado o compartilhamento da pasta `/var/www/html` entre os contêineres do WordPress e a máquina hospedeira, conforme solicitado. Nos testes com o comando `curl -I http://localhost/`, o cabeçalho `X-Upstream` alternou entre diferentes endereços IP (`172.18.0.3:80`, `172.18.0.4:80` e `172.18.0.5:80`), comprovando que o balanceamento de carga está funcionando corretamente.
